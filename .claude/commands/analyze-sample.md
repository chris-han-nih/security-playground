# 보안 코드 샘플 분석

대상 파일: $ARGUMENTS

## 분석 프로세스

### 1단계: 파일 읽기 및 분류

파일을 읽고 다음을 판단:
- 취약점 예제인지, 안전한 패턴인지, 테스트 코드인지
- 프로그래밍 언어
- 관련 CWE 번호 (있는 경우)

### 2단계: 코드 분석

**취약점 예제인 경우**:
1. 취약점 유형 식별
2. 취약점이 발생하는 정확한 코드 라인 지정
3. 공격 시나리오 설명
4. 잠재적 영향 평가

**안전한 패턴인 경우**:
1. 어떤 취약점을 방어하는지 식별
2. 방어 메커니즘 설명
3. 모범 사례 준수 여부 확인

### 3단계: 참조 자료 검색

WebSearch를 사용하여 관련 정보 수집:
- OWASP 가이드라인
- CWE 상세 정보
- 언어별 보안 권장사항
