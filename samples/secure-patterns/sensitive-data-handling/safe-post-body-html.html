안전한 인증 패턴

이 페이지는 민감한 정보를 안전하게 전송하는 방법을 보여줍니다.

핵심: POST Body 또는 Authorization 헤더 사용

안전한 로그인 폼

[보안] POST 메서드 사용
민감한 정보는 URL이 아닌 요청 본문(Body)으로 전송됩니다.

보안 구현 포인트

  1. POST 메서드 사용: GET 대신 POST로 민감 데이터 전송
  2. JSON Body: 데이터를 요청 본문에 JSON 형식으로 포함
  3. Authorization 헤더: 토큰 인증 시 헤더 사용
  4. Referrer-Policy: 메타 태그로 Referer 노출 제한

코드 비교

취약한 방식 (GET)
/login?username=admin&password=secret
비밀번호가 URL에 노출됨
안전한 방식 (POST Body)
POST /login
Body: {"username":"admin","password":"secret"}
비밀번호가 URL에 노출되지 않음
